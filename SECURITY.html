Security

Reporting Security Issues

If you discover a security vulnerability in this project, please open a GitHub issue in this repository.

At this stage of the project, GitHub issues are the preferred reporting channel. Please describe the vulnerability clearly, including steps to reproduce where possible. Avoid including any sensitive details (such as live API keys or credentials) in the public issue.

Sensitive Files and Secrets

.env Files

Application secrets are stored in .env files and loaded at runtime. The .env file is listed in .gitignore and must never be committed to source control.

Do:

  • Use .env.example as a template — it contains placeholder values only.
  • Keep your real .env file local and untracked.
  • Rotate any key that you accidentally commit immediately.

Do not:

  • Commit .env or any file containing live secrets.
  • Share your .env file with others.
  • Hard-code secrets in source files.

GOOGLE_API_KEY

The GOOGLE_API_KEY environment variable enables optional Gemini AI tagging features. It is optional; leave it unset or empty to disable all AI features.

Safe handling guidance:

  • Store the key only in your local .env file.
  • Grant the key the minimum required permissions in the Google Cloud / AI Studio console.
  • Rotate the key if you suspect it has been exposed.
  • Do not set this key in shared or multi-user environments unless you fully understand the access implications.

Portable Deployments (prepare_portable_target.bat)

prepare_portable_target.bat creates a safe, clean .env on the deployment target by copying from .env.example. Your development .env (including any live GOOGLE_API_KEY) is never copied to the target drive.

The .env placed on the portable target contains only placeholder comments — the same content as .env.example. Users of the portable copy can add their own API key after deployment if they want AI auto-tagging features.

Dependency Security

  • Keep dependencies up to date. Run pip install --upgrade -r requirements.txt periodically or when a vulnerability is announced.
  • Review the output of pip audit (or a comparable tool) to identify known vulnerabilities in installed packages.
  • The bundled Python runtime on portable deployments should also be updated when security patches are released for Python 3.12.

Least-Privilege Principles

  • Run the application with the minimum operating-system permissions needed.
  • The SQLite database file and its parent folder should be writable only by trusted local users or processes that need to run the application.
  • Do not expose the application's HTTP port to public networks; it is designed to run on localhost or a trusted local network only.
