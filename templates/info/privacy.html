Privacy

Local-First Design

Embroidery Catalogue is designed as a local-first application. By default, all data is stored on your own machine (or the portable drive on which the app is deployed) and no information is transmitted to external servers.

Data Stored Locally

The following data is stored locally by the application:

Data Storage location
Embroidery design metadata (filename, dimensions, hoop size, category, designer, rating, tags, notes) Database (SQLite)
Imported file paths and filenames Database
Generated preview images (JPEG thumbnails) previews/ folder on disk
Project and project-item metadata Database
Application settings Database / environment variables

No usage analytics, telemetry, or crash reports are collected or transmitted.

Optional Gemini / Google AI Integration

Automated tag generation powered by Google Gemini is an optional, opt-in feature. It is off by default and requires you to:

  1. Obtain a Google Gemini API key from https://ai.google.dev/.
  2. Set the GOOGLE_API_KEY environment variable in your .env file.

When Gemini features are enabled, the following information may be sent to Google's servers:

  • The filename of the embroidery design being tagged.
  • A preview image (JPEG thumbnail) of the design.
  • A text prompt describing what tagging information is requested.

This data is processed by Google in accordance with Google's Privacy Policy and the Gemini API Terms of Service.

You are responsible for reviewing and accepting Google's privacy policy and terms of service before enabling this feature. Do not enable AI tagging for files that contain sensitive or personal information.

API Key Handling

Your GOOGLE_API_KEY is stored in your local .env file and loaded at runtime. It is never logged, displayed in the UI, or transmitted anywhere other than to Google's API endpoints.

Warning: Do not commit your .env file to source control, and do not share copies of the application (including portable deployments) that contain a live API key. See SECURITY.md for guidance on safe secret handling.

Portable Deployments

When the application is copied to a deployment target using prepare_portable_target.bat, any .env file present in the source folder is not automatically excluded from the copy. If your .env file contains a live GOOGLE_API_KEY, that key will be present on the portable copy. Ensure you understand the implications before distributing the drive to others.
