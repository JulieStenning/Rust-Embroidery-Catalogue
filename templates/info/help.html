🔍 Search

Use the general search box on the Browse page to find designs by keyword. The search checks the filename, notes, designer, source, and tags.

Quoted phrases

Use double quotes to match an exact phrase: "cross stitch"

Exclude terms

Prefix a word with - to exclude it: rose -applique

OR searches

Use OR between terms: rose OR tulip

Filename wildcards

In the Filename filter, use * for any text and ? for a single character: rose*.jef

Unverified only

Tick Unverified only to show designs that still need a tag or metadata review.

Quick search vs. filters

Use the general search box for a fast keyword search. Expand Additional filters for precise control over all-words, exact phrase, any-words, none-words, designer, hoop, and more.

📥 Importing

Use Import to scan one or more folders and their sub-folders to add embroidery files to the catalogue.

Choosing folders

Enter folder paths directly, click Browse…, or use Add another folder to import from several locations in one run. On Windows, the native picker can select multiple folders in one go. Sub-folders are scanned automatically.

Review and metadata step

After scanning, the files found are grouped by folder. You can choose which files to import and set the Designer and Source for each folder before continuing.

Designer and source choices

For each folder, you can let the app keep the inferred value, choose an existing Designer or Source, create a new one during import, or leave it blank. When importing multiple folders, you can also apply the same choice to all folders first and then override individual folders if needed.

Tag check before import

When you click Continue, the app pauses before importing. On the very first import it asks you to review your tags first; on later imports you can choose whether to review tags or import straight away. New tags must be saved as Image or Stitching, and the ✓ button next to the group selector saves any change.

AI tagging notice

The Before You Import step shows a notice about AI tagging status. If no API key is configured, a blue notice confirms that only Tier 1 keyword tagging will run. If a key is present, an amber notice shows the cost/quota warning and your current Tier 2/Tier 3 settings. Click Change in Settings to adjust the AI tier options before continuing.

Error files and large scans

Files shown with errors are unreadable or unsupported and will be skipped automatically. For very large imports, the review step may show summary counts instead of every individual row for better speed.

🤖 AI Tagging

Optional Google AI tagging can suggest design-type tags for imported embroidery files. It only needs a Google API key if you want to use the Gemini-powered tagging tiers.

Getting an API key

Create a key in Google AI Studio, then open Settings to save it in the app.

Enabling AI tagging tiers

In Settings, tick Run Tier 2 automatically during import to send filenames to Gemini for tag suggestions, and/or Run Tier 3 automatically during import to send preview images to Gemini Vision. Tier 1 keyword tagging always runs for free and does not require a key. If neither tier is enabled, imports run Tier 1 only even when a key is present.

Batch size and delay for large collections

Set an optional AI tagging batch size in Settings to limit how many designs are sent to Gemini per import run. Set the optional delay (seconds between API calls) to avoid rate-limit errors. Both can also be overridden per-action on the Tagging Actions page. Leave them blank to use the defaults.

In-app Tagging Actions

The Admin → Tagging Actions page lets you run AI tagging on existing designs without the command line. Choose from three actions: Tag only untagged (safe, never touches verified tags), Tag untagged and unverified (skips manually verified designs), and Re-tag ALL designs (overwrites everything — requires confirmation). Each action lets you choose tiers, set a batch size, and adjust the delay per run. The same page also includes a Local stitching backfill option for your offline example-image matcher.

Rate-limit errors (429 Too Many Requests)

If you see 429 errors, increase the delay in Settings or in the Tagging Actions form. You can also use a smaller batch size to spread calls across multiple runs. For very large libraries, consider running the CLI scripts overnight: python auto_tag.py --delay 6.0 --limit 200.

Potential costs

Keyword-only tagging is free. Gemini also has a free tier for lighter use, while larger collections may incur charges. A February 2026 estimate found Tier 3 on 4,000 images cost about $0.33 on the paid tier; this may have changed. See the current pricing page for the latest rates.

Import warnings

The Before You Import step shows a blue notice if no key is configured (Tier 1 only), or an amber cost notice if a key is present showing which tiers are enabled. You can click Change in Settings from that page to adjust tiers before continuing.

Full guide

Read the AI Tagging Guide for setup steps, security notes, free-tier limits, and example CLI usage.

🏷 Tagging Actions

The Admin → Tagging Actions page lets you run AI tagging on your existing designs without the command line.

Available actions

  • Tag only untagged designs — safe to run at any time; never overwrites verified tags.
  • Tag untagged and unverified designs — also processes designs with auto-suggested but unverified tags.
  • Re-tag ALL designs — overwrites every design including verified ones; requires confirmation.

Choosing tiers

Tier 1 (keyword matching) is always included. Tiers 2 and 3 are available when a Google API key is configured. Select the tiers you want for the current run.

Batch size and delay

You can limit the run to a specific number of designs and override the delay between API calls. Increase the delay if you see 429 Too Many Requests errors. Leave the fields blank to use the values saved in Settings.

Local stitching backfill

The Local stitching backfill action analyses the actual embroidery pattern geometry (via StitchIdentifier) to detect stitch types. It only updates the stitching tag group and skips designs whose tags you have already verified.

New user tip

If you are new to AI tagging, use Tag only untagged designs with a small batch size first, review the results, then run again for more designs.

📁 Projects

Projects let you group designs for a planned embroidery task.

What projects are for

A project is a named collection of designs you plan to stitch together — for example a set of Christmas designs or a quilt block series. Use the description field for optional notes, goals, or deadlines.

Adding designs

You can add designs one at a time from a design's detail page using the Add to Project selector at the bottom.

Adding several designs at once

On the Browse page, tick multiple designs and use the bottom bar's project selector between Verify selected and Clear selection, then click Add to project.

Printing

Each project has a print view so you can produce a reference sheet of all designs in the project — useful for planning and taking to the machine.

🛠 Maintenance

The Orphans tool (accessible via the nav bar) finds design records in the database whose files are no longer present on disk.

What orphaned records are

An orphaned record is a design that exists in the database but whose file cannot be found in the catalogue's managed storage folder. This usually happens when files are removed from `data\MachineEmbroideryDesigns`, renamed outside the app, or a backup was restored incompletely.

Deleting orphans

Deleting an orphan removes the database record only. It does not delete the actual design file from disk (if it still exists somewhere). Use this to clean up stale records after reorganising your file library.

When to use this carefully

Before deleting, verify that the file really is gone or has moved. Because the catalogue now uses managed portable storage, drive-letter changes should not normally cause orphans on their own.

🔧 Troubleshooting

Missing folder / changed drive letter

If many designs show as orphaned after moving the app to a new drive, make sure the entire `data\` folder moved with it. No separate path reconfiguration should be needed.

Import scan finds nothing

Check that the folder path is correct and that it contains supported embroidery formats. The app supports a broad range of pyembroidery-readable formats including .jef, .pes, .hus, .vp3, .dst, .exp, .sew, .u01, and many more; .art support is limited. If using a network share, ensure it is connected and accessible before scanning.

Files missing after import

If files were imported and then moved or deleted, they will appear as orphans. Use the orphan maintenance tool to clean up the stale records, then re-import from the new location.

Managed storage location

Imported files are stored inside the catalogue at `data\MachineEmbroideryDesigns`. If a design cannot be opened, check that the file still exists there and that the `data\` folder was copied or restored completely.

Still stuck?

Check that the application is running with sufficient permissions to read your design folder. If the problem persists, review the application logs for more detail.
