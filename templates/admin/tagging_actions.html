4. Stitch Type Analysis - Detect embroidery styles directly from your design files.

How it works: Analyses your embroidery files to identify specific stitch types.

Targeting: Only processes unverified designs to ensure your catalogue stays accurate.

5. Threads and colours - Automatically retrieve technical details from your embroidery files.

How it works: Scans your files to identify the total stitch count, the number of individual colours, and the number of thread changes required.

Targeting: Only updates designs where this information is currently missing.

Refresh all thread and colour data - Updates the stitch and colour counts for every design, overwriting any existing information.

6. Images - Generate visual previews and sizing for your collection.

How it works: Reads each embroidery file to create a preview image, calculate the width and height, and set the hoop.

Targeting: Only processes designs where these images or measurements are currently missing.

Delete all existing images?

After deletion, the image backfill will run and regenerate previews only for designs where images are missing — making subsequent runs much faster if the process is interrupted.

CLI tools (advanced / overnight runs)

For very large collections or scheduled runs, the command-line scripts remain available. Activate the virtual environment and run from the project root: 

REM Tag only designs with no tags yet (default)
python auto_tag.py

REM Re-tag ALL designs (overwrites existing tags)
python auto_tag.py --redo

REM Keyword matching only — no Gemini calls
python auto_tag.py --tier1-only

REM Run Tier 2 + vision AI (Tier 3) on still-untagged designs
python auto_tag.py --tier3

REM Tag only unverified designs (skip verified)
python auto_tag.py --skip-verified

REM Tag unverified designs only — also run Tier 3 on still-untagged
python auto_tag.py --skip-verified --tier3

REM Limit to first 50 designs (for testing)
python auto_tag.py --limit 50

REM Slow down to avoid 429 rate-limit errors
python auto_tag.py --delay 6.0

REM Preview without writing — dry run
python auto_tag.py --dry-run
